SWEO AI — Production Widget

Chat Widget Test

The chat widget loads configuration from your tenant settings, supports AI conversations with RAG, human agent handoff, queue management, and real-time message polling.

Core Features

💬 AI Conversation

Click the chat bubble to start. The AI uses your knowledge base via RAG to answer questions, with source citations.

✉️ Email Verification

If requireEmail is enabled, users enter their name and email before chatting — stored for all future messages.

🎨 Dynamic Branding

Brand color, bot name, position, and welcome message load from /api/widget/config. No widget rebuild needed.

Handoff & Human Agents

🙋 Talk to a Person

Customers can click "Talk to a person" to request a human agent. The AI summarizes the conversation and places them in queue.

📋 Queue & Status

While waiting, customers see their queue position. The widget polls /api/chat/queue-status in real time.

👤 Agent Messages

Human agent replies show a green avatar and "Agent" badge. System messages (close, handoff) appear as centered banners.

Lifecycle

⭐ CSAT Feedback

After each AI resolution, a 1–5 star rating widget appears. Feedback is stored via /api/conversations/csat.

🔄 Session Restore

Conversations persist across page reloads via sessionStorage. Status, messages, and polling resume automatically.

The widget only needs data-api-key and data-api-url — everything else is loaded from the server.
